Introduction to Finetuning

What is Supervised Finetuning?

Supervised finetuning (SFT) is the process of adapting a pre-trained language model to perform specific tasks by training it on carefully curated examples.

Why Finetune?

  • Specialize models for specific tasks
  • Improve performance on domain-specific problems
  • Reduce hallucinations and enhance reliability
  • Create custom AI assistants

Pre-training vs Finetuning

Pre-training: Learning general language understanding from vast amounts of text

Finetuning: Learning specific tasks from curated examples

02

Choosing Your Base Model

Model Selection Criteria

  • Model Size

    Balance between performance and resource requirements

    • Small (1-3B parameters): Fast, limited capability
    • Medium (7B parameters): Good balance
    • Large (13B+ parameters): Best performance, resource-intensive
  • License

    Choose based on your use case:

    • Llama 2: Meta AI's license
    • Mistral: Apache 2.0
    • TinyLlama: MIT License

Popular Base Models

Model Size License Best For

Mistral 7B 7B Apache 2.0 Production

04

Introduction to LoRA

What is LoRA?

Low-Rank Adaptation (LoRA) is a technique that significantly reduces the memory and computational requirements of finetuning by only training a small set of adaptable parameters.

LoRA Parameters

# Example LoRA configuration
lora_r: 8          # Rank of LoRA update matrices
lora_alpha: 16     # Alpha scaling factor
lora_dropout: 0.05 # Dropout probability
target_modules:    # Layers to apply LoRA
  - q_proj
  - v_proj

Training Process

  1. Setup Environment

    pip install transformers peft datasets

  2. Prepare Training Script

    from peft import LoraConfig, get_peft_model

model = AutoModelForCausalLM.from_pretrained(
    "mistralai/Mistral-7B-v0.1"
)
lora_config = LoraConfig(
    r=8, alpha=16,
    target_modules=["q_proj", "v_proj"]
)

Benefits of LoRA

  • 95% less training memory
  • Faster training times
  • Small adapter size
  • Comparable performance to full finetuning
